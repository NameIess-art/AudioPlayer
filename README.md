# Nameless Audio

Nameless Audio 是一个面向 Android 本地音频库、ASMR、多会话播放、字幕和睡眠计时场景的 Flutter 播放器。  
界面、状态管理和数据持久化由 Flutter 承载，后台播放、MediaSession、通知、定时器保活、文件扫描和系统权限跳转由 Android 原生链路兜底。

当前版本：`0.8.82+882`  
最新发布页：[v0.8.82](https://github.com/NameIess-art/nameless-audio/releases/tag/v0.8.82)  
许可证：[MIT](https://github.com/NameIess-art/nameless-audio/blob/main/LICENSE)  
隐私说明：[PRIVACY.md](https://github.com/NameIess-art/nameless-audio/blob/main/PRIVACY.md)

> 注意：自 `v0.8.0` 起，Android `applicationId` 已从 `com.example.music_player` 更改为 `com.nameless.audio`。旧包名版本不能直接覆盖安装到新版本上，升级前请先备份并重新安装。

## 主要功能

### 播放

- **多会话播放**：可同时创建多个独立播放会话，每个会话都能单独控制播放、暂停、进度、循环模式、声道互换和音量。
- **原生后台播放队列**：非单曲循环模式下会把当前文件夹或跨文件夹队列下沉到 Android Media3 / ExoPlayer，息屏和后台状态下也能稳定续播、循环和随机播放。
- **后台播放稳定性**：播放前台服务、MediaSession、WakeLock、原生保活服务和精确定时协同工作，优先保证长时间后台与息屏播放。
- **音频焦点管理**：原生处理系统音频焦点，支持来电或其他媒体抢占后的暂停与恢复。
- **循环与随机模式**：支持单曲循环、当前文件夹顺序、当前文件夹随机、跨文件夹顺序、跨文件夹随机。
- **音量增益**：每个会话独立音量，支持 `0% - 200%`；超过 `100%` 的部分通过 Android `LoudnessEnhancer` 实现增益放大。
- **左右声道互换**：可按会话开启左右声道互换，适合特定 ASMR 内容。

### 播放列表与会话详情

- **会话列表**：支持拖拽排序、左滑移除、批量暂停全部、清空全部。
- **底部播放卡片**：可选开启常驻底部播放卡片，并同步显示封面、进度、字幕和播放状态。
- **会话详情页**：以封面作为模糊背景，提供主封面、进度条、快进快退、循环模式、音量、计时器和字幕控制。
- **轨道切换器**：可在会话详情页直接切换当前文件夹 / 分组内的其他音轨，无需新建会话。
- **封面预热**：左右切换会话时会预热相邻会话封面，减少详情页切换时的闪烁。
- **字幕位置持久化**：全局悬浮字幕的位置按会话保存，重新打开详情页后可恢复。

### 音频库

- **层级媒体库导航**：根据文件树自动生成层级节点，适合管理包含大量子文件夹的本地媒体库。
- **三种导入方式**：支持导入单个文件夹、导入整套曲库、导入多个独立文件。
- **原生优先导入**：Android 上优先使用系统文档选择器（SAF）导入 `content://` 资源并持久化授权；不可用时回退到旧文件选择流程。
- **扫描进度卡片**：扫描时显示当前文件夹、已发现数量、重复数量和失败数量。
- **下拉刷新**：重新扫描所有监听目录，自动补齐新增文件，并移除磁盘中已删除的文件。
- **搜索与高亮**：支持按名称搜索并高亮结果，同时显示匹配数量。
- **自然排序与拖拽排序**：文件 / 文件夹名称按自然顺序排序；根目录支持手动拖拽调整顺序。
- **左滑操作**：支持左滑移除文件夹或音频，也支持左滑打开详细信息。
- **分类浏览**：支持按全部、标签、声优、社团浏览音频。
- **分类项长按复制**：标签、声优和社团项支持长按复制名称。
- **自动预填 RJ 号**：导入文件夹时会尝试从文件夹名称中提取 RJ 编号，自动预填到详细信息。

### 曲库管理

- **曲库编辑页**：完整展示已导入曲库结构，支持排除 / 恢复文件夹或单曲，并在同一页内搜索过滤条目。
- **排除状态持久化**：曲库排除状态持久化到 SQLite，重启后可自动恢复。
- **刷新后恢复**：被排除的条目不会在刷新时自动加回；重新导入或恢复时会按规则重新同步。
- **重命名联动**：重命名已导入文件夹后，曲库条目、封面、备份和播放列表关联信息会一起更新。

### 作品详细信息与元数据

- **作品详细信息页**：支持查看和编辑文件 / 文件夹名称、RJ 号、作品标题、社团、声优、标签等信息。
- **DLsite 元数据获取**：可按 RJ 号或标题从 DLsite 获取元数据，并在确认页中审核后再写入。
- **按作品标题重命名**：支持把单个文件或文件夹直接重命名为作品标题。
- **文件夹封面选择器**：在文件夹详细信息中，若存在候选图片，会在“文件夹名称”上方显示封面选择区。
- **滑动后延迟应用**：左右滑动切换候选图片后，若 1 秒内没有继续切换，则自动把当前图片设为文件夹封面。
- **当前封面起始定位**：每次打开详细信息页时，封面选择器都会从当前已生效的封面图片开始显示。
- **元数据备份**：详细信息会自动备份到同目录下的 `nameless-audio.json`，数据库丢失时可自动恢复。

### 封面

- **自动封面发现**：优先查找常见文件名，如 `cover`、`folder`、`front`、`album`、`artwork`、`poster`。
- **视频帧封面**：视频媒体可提取视频帧作为封面候选。
- **文件夹封面手动选择**：通过文件夹详细信息页中的封面选择区切换并保存封面。
- **封面全局同步**：手动设置封面后，音频库文件夹卡片、播放列表音频卡片、底部播放卡片和播放详细页会同步更新。

### 字幕

- **格式支持**：支持 `.srt`、`.ass`、`.ssa`、`.vtt`、`.lrc`。
- **会话内字幕**：会话详情页可内联显示当前字幕文本。
- **全局悬浮字幕**：开启后可跨页面 / 后台显示字幕，关闭字幕时会同步关闭全局显示。
- **样式自定义**：支持字体、字号、字体颜色、背景颜色、透明度、模糊强度和边框深度调整，并持久化保存。
- **卡片与通知字幕**：底部播放卡片和通知可同步显示当前字幕文本。

### 睡眠计时器

- **两种启动模式**：支持手动倒计时（立即开始）和播放触发倒计时（等播放开始后再计时）。
- **到点暂停**：倒计时结束后通过 Android 原生链路暂停所有会话。
- **自动恢复**：可设置在指定本地时间自动恢复被计时器暂停的会话。
- **可靠性检查卡片**：计时器页会检查通知权限、精确定时权限和后台运行条件，并提供快捷跳转修复入口。
- **紧凑模式与胶囊状态**：播放列表顶部显示计时胶囊，计时器页支持配置态与倒计时态切换。

### 通知与后台权限

- **多会话统一通知**：支持单会话和多会话通知样式，并提供播放、暂停、上一首、下一首、恢复通知和清空通知等控制。
- **通知控制开关**：设置页可以关闭富播放通知样式；播放中仍保留系统前台通知以保证后台稳定。
- **权限前置提示**：通知、精确定时、后台运行、悬浮窗和应用内安装等权限会先提示再跳转设置，返回应用后继续原操作。

### 视频转音频

- **格式与码率**：支持输出 `mp3`、`aac`、`ogg`、`wav`、`flac`；`wav` 和 `flac` 不使用码率参数。
- **进度与取消**：显示实时转换进度，可在转换中途取消。
- **输出防覆盖**：自动避开与现有文件重名的输出路径。

### 应用内更新

- **GitHub Releases 检查更新**：可在设置页检查最新版本、下载并触发系统安装流程。
- **安装权限续接**：如缺少“允许安装未知来源应用”权限，会先提示并跳转设置，返回后继续安装。

### 外观与偏好

- **深色模式**：支持跟随系统或手动切换。
- **多语言**：支持中文、日文、英文。
- **多线程播放开关**：可控制是否允许同时存在多个活跃播放会话。
- **添加后自动播放**：新建会话后可自动开始播放。
- **清理临时缓存**：设置页支持清理 SAF 导入产生的临时缓存。
- **Tab 回顶**：点击当前激活的 Tab 图标时，列表自动滚回顶部。

## 导入方式

在音频库页右上角 `+` 菜单中提供三种导入方式：

| 方式 | 适合场景 | 会导入什么 | 后续刷新行为 |
|---|---|---|---|
| 导入文件夹 | 只想监听一个固定目录 | 扫描该目录及其子目录中的受支持媒体文件，并把这个目录加入监听列表 | 下拉刷新时会重新扫描这个目录 |
| 添加曲库 | 有一个大目录，且每个一级子目录都是一套内容 | 把根目录下一层子目录分别加入监听，根目录中散落的媒体文件也会一起导入 | 下拉刷新时会重新同步整套曲库结构，并保留排除规则 |
| 导入文件 | 想临时加入几个独立媒体文件 | 只导入手动选中的文件，不绑定到监听文件夹 | 不参与文件夹扫描，直到手动移除 |

### Android 上的实际导入策略

- 导入文件夹 / 添加曲库：优先使用系统文档树选择器（SAF），拿到持久化读取授权；不可用时再回退到旧流程。
- 导入文件：优先使用系统文件选择器（支持多选），直接保留 `content://` 访问授权；不支持时回退到旧文件选择器。
- 很多机型即使没有完整文件管理权限，也可以成功导入和播放用户明确选择的文件或目录。

| 音频 | `flac`、`wav`、`mp3`、`m4a`、`aac`、`ogg`、`opus`、`3gp` |

## 权限说明

| 权限 | 是否必须 | 用途 |
|---|---|---|
| `READ_MEDIA_AUDIO` / `READ_EXTERNAL_STORAGE` | 视机型而定 | 兼容直接文件系统扫描与旧式文件选择流程 |
| `MANAGE_EXTERNAL_STORAGE` | 否 | 仅用于需要完整文件系统访问的机型 / 路径，不是 SAF 导入的前提 |
| `POST_NOTIFICATIONS` | 推荐 | 播放通知、后台控制、状态提示 |
| `SYSTEM_ALERT_WINDOW` | 仅字幕全局显示需要 | 允许悬浮字幕跨页面 / 后台显示 |
| `FOREGROUND_SERVICE` / `FOREGROUND_SERVICE_MEDIA_PLAYBACK` | 是 | 后台与息屏播放 |
| `WAKE_LOCK` | 是 | 降低息屏后 CPU 过早休眠导致的播放中断概率 |
| `REQUEST_IGNORE_BATTERY_OPTIMIZATIONS` | 推荐 | 引导用户允许后台运行 / 忽略电池优化 |
| `SCHEDULE_EXACT_ALARM` / `USE_EXACT_ALARM` | 推荐 | 提升定时暂停与自动恢复的可靠性 |
| `RECEIVE_BOOT_COMPLETED` | 推荐 | 系统重启或应用更新后恢复计时器状态 |
| `REQUEST_INSTALL_PACKAGES` | 仅应用内更新需要 | 下载新 APK 后触发系统安装流程 |
| `INTERNET` | 仅更新检查和 DLsite 信息获取需要 | 检查 GitHub Releases、下载更新、获取 DLsite 元数据 |

## 下载

从 [GitHub Release v0.8.82](https://github.com/NameIess-art/nameless-audio/releases/tag/v0.8.82) 下载适合设备 CPU 架构的 APK：

|---|---|

## 项目结构

```text
lib/
  i18n/                         中 / 日 / 英多语言文案
  models/                       MusicTrack、LibraryNode、LibraryEntry、PlaybackMode、作品详情模型
  providers/                    AudioProvider 门面与功能拆分
  screens/                      音频库、播放列表、计时器、设置、视频转音频、DLsite 信息确认
  services/                     SQLite、Native 桥接、更新、字幕、通知、权限控制、DLsite 元数据
  theme/                        主题与深色模式
  widgets/                      通用组件与业务组件
android/app/src/main/kotlin/    原生播放、通知、扫描、计时闹钟、保活服务、字幕悬浮窗
third_party/audio_service/      项目内维护的 audio_service fork
test/                           数据库、Provider、通知、计时器、播放桥接等测试
```

### Release 构建

Release 构建需要签名密钥。仓库当前保留了本地验证用的回退逻辑：未配置 `key.properties` 时，会使用 debug signing 生成 release 模式 APK；正式发布前建议配置 release keystore。

```bash
flutter build apk --release --split-per-abi
```

该命令会生成三个 APK：

- `app-arm64-v8a-release.apk`
- `app-armeabi-v7a-release.apk`
- `app-x86_64-release.apk`

## 发行说明 v0.8.82

- **熄屏长播稳定性增强**：原生播放服务现在会区分“系统临时丢失音频焦点”和“用户主动暂停”，避免熄屏后被误判为真正停播。
- **服务重启后自动续播**：`NativePlaybackService` 被系统回收后，会从原生持久化快照中恢复仍应继续播放的会话、队列、循环和随机状态。
- **长期后台运行提示补强**：开始播放时若系统仍限制后台运行，会再次提示用户开启电池优化白名单，降低长时间熄屏后被系统限制的概率。
- **README 同步更新**：版本、下载入口和本次稳定性改动说明已经同步到文档。
