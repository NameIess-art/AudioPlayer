# Nameless Audio

Nameless Audio 是一个面向 Android 本地媒体库、多会话 ASMR 播放、长音频、字幕和睡眠计时场景的 Flutter 播放器。界面和业务状态由 Flutter 承载，后台播放、通知控制、息屏保活、定时暂停/自动恢复、媒体扫描与系统权限跳转由 Android 原生能力兜底。

当前版本：`0.8.8+880`<br>
最新发布页：[v0.8.8](https://github.com/NameIess-art/nameless-audio/releases/tag/v0.8.8)<br>
许可证：[MIT](https://github.com/NameIess-art/nameless-audio/blob/main/LICENSE)<br>
隐私说明：[PRIVACY.md](https://github.com/NameIess-art/nameless-audio/blob/main/PRIVACY.md)

> 注意：自 `v0.8.0` 起，Android `applicationId` 已从 `com.example.music_player` 变更为 `com.nameless.audio`。旧包名版本无法直接覆盖安装到新版本上，升级前请先备份并重新安装。

## 主要功能

### 播放

- **多会话播放**：可同时创建多个播放会话；每个会话独立控制播放/暂停、进度、音量（0–200%）、循环模式和左右声道互换。
- **原生后台播放队列**：非单曲循环模式会把当前文件夹或跨文件夹队列下沉到 Android Media3/ExoPlayer，息屏和后台状态下也能稳定续播、循环和随机播放。
- **Android 后台保活**：播放通知、前台服务、WakeLock、精确闹钟、原生播放服务和保活服务协同工作，提升几十小时级别的后台与息屏播放稳定性。
- **精准音频焦点管理**：自动处理 Android 系统音频焦点，支持来电暂停、其他媒体抢占后自动暂停及焦点恢复；禁用 ExoPlayer 内置焦点管理，防止系统提示音等短暂焦点丢失导致播放中断。
- **循环模式**：单曲循环、当前文件夹顺序、当前文件夹随机、跨文件夹顺序、跨文件夹随机。
- **音量增益**：每个会话独立音量，支持 0–200% 范围；超过 100% 时通过 Android `LoudnessEnhancer` 实现增益放大（约 +6 dB）。
- **声道互换**：可对单个会话开启左右声道互换，适合特定 ASMR 内容。
- **轨道切换器**：在会话详情页可打开当前文件夹/分组的曲目列表，直接切换到同组内的其他曲目，无需新建会话。

### 播放列表

- **会话列表**：支持拖拽排序、左滑移除、批量暂停全部、清空全部。
- **状态指示**：会话卡片上内联显示字幕（🔤）和声道互换（↔）状态图标。
- **音量快捷控制**：会话卡片上的音量按钮弹出垂直滑块浮层，点击百分比数字可输入精确数值（0–200%）。
- **计时胶囊**：计时器激活时，播放列表头部显示实时倒计时胶囊；计时结束且设置了自动恢复时，胶囊切换为自动恢复倒计时；点击可跳转到计时器页。

### 会话详情页

- **封面背景**：详情页以封面图作为全屏模糊背景，叠加颜色遮罩。
- **进度条**：显示播放进度和缓冲进度双轨，支持 5 秒快进/快退。
- **字幕面板**：详情页内联显示当前字幕文本，独立于全局悬浮窗。
- **封面预缓存**：左右滑动切换会话时，相邻会话的封面会提前预缓存，保证切换流畅。
- **字幕位置持久化**：全局悬浮字幕的位置按会话保存，重新打开详情页时自动恢复。

### 音频库

- **层级媒体库导航**：支持文件夹分层浏览，自动根据文件树结构生成导航节点，方便管理包含数千个文件的复杂媒体库。
- **三种导入方式**：支持导入单个文件、导入单个文件夹、导入整个资料库（父目录下的多一级子目录）。
- **原生优先导入**：Android 上优先使用系统文档选择器（SAF）导入 `content://` 资源并持久化读取授权；不可用时保留旧式文件选择回退。
- **扫描进度卡片**：扫描时浮动显示当前文件夹、已发现数量、重复数量和失败数量。
- **加载骨架屏**：库初始化时显示脉冲占位骨架，避免空白闪烁。
- **下拉刷新**：重新扫描所有监听目录，自动补齐新增文件，并移除已从磁盘删除的文件（后台 isolate 异步检查）。
- **搜索**：支持搜索高亮、匹配数量提示。
- **排序**：根目录支持拖拽排序；新导入或刷新同步出的根目录排在列表最上方。
- **左滑操作**：支持左滑移除，以及左滑打开作品详情。
- **分类浏览**：可按全部、标签、声优、社团浏览音频；分类元素支持长按复制文本；分类视图也支持下拉刷新。
- **自然排序**：文件和文件夹名称使用自然排序（如"Track 2"排在"Track 10"之前）。
- **自动预填 RJ 号**：导入文件夹时，自动从文件夹名称中提取 RJ 号并预填到作品详情。

### 曲库管理

- **曲库结构持久化**：每个曲库的文件夹、子结构和音频条目写入 SQLite，记录使用/排除状态；后续新增内容可通过刷新同步。
- **编辑曲库**：完整展示曲库结构，支持排除/恢复文件夹或单曲；父文件夹被排除时子项同步显示已排除，恢复父文件夹后子结构同步恢复；编辑页内置搜索栏，可在曲库内过滤条目。
- **排除状态持久化**：排除状态以 SQLite 为权威来源，重启应用后自动恢复，不依赖 SharedPreferences 缓存。

### 作品详情与元数据

- **作品详情编辑**：支持编辑作品标题、RJ 号、社团、声优、标签等信息。
- **DLsite 元数据**：可按 RJ 号或标题从 DLsite 获取元数据；确认页显示 RJ 号，可保存封面到文件夹。
- **重命名**：在作品详情页可将文件夹或文件重命名为作品标题，直接修改磁盘上的实际名称。
- **元数据备份**：作品详情自动备份到 `nameless-audio.json` 文件——曲库根目录下保存为单个 JSON 对象，单独导入的文件则以数组格式保存在同目录下（多个文件共享一个备份文件，按路径区分）；数据库丢失时可自动从备份恢复。

### 封面

- **自动封面发现**：优先查找常见文件名，如 `cover`、`folder`、`front`、`album`、`artwork`、`poster`。
- **视频帧封面**：对视频媒体可提取视频帧作为封面候选。
- **自定义封面**：长按详情页封面进入选择模式，左右滑动浏览根目录中的图片并确认替换；自定义封面同步刷新详情页、音频库卡片、播放列表卡片和底部播放卡片。

### 字幕

- **格式支持**：支持 `.srt`、`.ass`、`.ssa`、`.vtt`、`.lrc`。
- **会话内字幕**：详情页内联显示当前字幕文本。
- **全局悬浮窗**：启用"字幕全局显示"后，字幕悬浮窗跨页面/后台显示；关闭字幕时同步关闭全局显示。
- **样式自定义**：字体族（7 种选项，含宋体、楷体、黑体等）、字号、字体颜色（RGB 滑块 + 数值输入）、背景模糊、背景透明度、背景颜色（RGB 滑块）、边框深度；所有设置持久化保存。
- **底部卡片与通知字幕**：底部播放卡片和通知可显示当前字幕文本。

### 睡眠计时器

- **两种模式**：手动倒计时（立即开始）和播放触发倒计时（等待播放开始后才计时）。
- **到点暂停**：计时结束后通过 Android 原生链路暂停所有会话。
- **自动恢复**：可设置在指定本地时间自动恢复被计时器暂停的会话；计时结束后胶囊显示自动恢复倒计时。
- **计时器可靠性卡片**：配置计时器时显示通知权限、精确闹钟、电池优化三项状态，并提供内联修复按钮。
- **计时结束后重置**：无自动恢复时，计时结束后计时器自动恢复到未配置状态；自动恢复播放后同样重置。
- **紧凑模式**：计时器可嵌入播放列表头部，支持配置页与倒计时页之间的动画切换。

### 通知控制

- **单/多会话通知**：支持单会话和多会话通知样式，提供播放/暂停、上一首、下一首、恢复通知、清空通知等控制。
- **通知控制开关**：可在设置中关闭富通知样式，关闭后不显示任何通知（包括最小前台通知）。

### 视频转音频

- **格式与码率**：支持 `mp3`、`aac`、`ogg`、`wav`、`flac` 输出；`wav` 和 `flac` 不使用码率参数；格式和码率选择持久化保存。
- **转换控制**：显示实时进度，可随时取消；输出文件自动避免覆盖已有文件。

### 应用内更新

- 可在设置页检查 GitHub 最新版本，下载并安装新版 APK。
- 下载前检查"允许安装未知来源应用"权限；未授权时先弹提示再跳设置，返回后自动继续安装。

### 外观与偏好

- **深色模式**：跟随系统或手动切换。
- **语言**：中文、日文、英文。
- **底部播放卡片**：可开关底部常驻播放卡片。
- **添加后自动播放**：新建会话时自动开始播放。
- **多线程播放**：开启后可同时播放多个会话；关闭后始终只保留一个播放会话。
- **临时缓存清理**：可在设置页清理 SAF 导入时产生的临时文件缓存。
- **Tab 回顶**：点击当前激活的 Tab 图标，列表自动滚动回顶部。

### 权限前置提示

涉及通知、未知来源安装、悬浮窗、后台运行、精确定时等权限的操作，会先提示再跳设置；用户返回应用后会自动继续原操作。

## 导入方式

在音频库页右上角 `+` 菜单中有三种导入方式：

| 方式 | 适合场景 | 会导入什么 | 后续刷新行为 |
|---|---|---|---|
| 导入文件夹 | 你只想监听一个固定目录 | 扫描该目录及其子目录中的受支持媒体文件，并把这个目录加入监听列表 | 下拉刷新时会重新扫描这个目录，新增根节点优先显示在顶部 |
| 添加曲库 | 你有一个"大目录"，里面每个一级子目录都是一套内容 | 会把所选根目录下的一级子目录分别加入监听；根目录里散落的媒体文件也会一并导入 | 下拉刷新时会重新扫描整套资料库结构，补齐新增文件夹/文件，并保留排除规则 |
| 导入文件 | 你只想临时或零散加入几个媒体文件 | 只导入手动选中的文件，标记为"单曲"，不绑定到某个监听文件夹 | 不参与文件夹扫描；文件仍保留在库中，直到你手动移除 |

### Android 上的实际导入策略

- 导入文件夹/添加曲库：优先使用系统文档树选择器（SAF），拿到持久化读取授权；如果设备 ROM 的文档界面不可用，会回退到旧的文件夹选择流程。
- 导入文件：优先使用系统文档文件选择器（支持多选），直接保留 `content://` 访问授权；部分设备不支持时会回退到旧文件选择流程。
- 很多机型上即使没有完整文件管理权限，也可以成功导入和播放用户明确选择的文件或目录。

## 权限说明

| 权限 | 是否必须 | 用途 |
|---|---|---|
| `READ_MEDIA_AUDIO` / `READ_EXTERNAL_STORAGE` | 视机型而定 | 兼容直接文件系统扫描与旧式文件选择流程 |
| `MANAGE_EXTERNAL_STORAGE` | 否 | 仅用于需要完整直接文件系统访问的机型/路径；不是 SAF 导入的前提 |
| `POST_NOTIFICATIONS` | 推荐 | 播放通知、后台控制、状态提示 |
| `SYSTEM_ALERT_WINDOW` | 仅字幕全局显示需要 | 允许悬浮字幕跨页面/后台显示 |
| `FOREGROUND_SERVICE` / `FOREGROUND_SERVICE_MEDIA_PLAYBACK` | 是 | 后台与息屏播放 |
| `WAKE_LOCK` | 是 | 降低息屏后 CPU 过早休眠导致播放中断的概率 |
| `REQUEST_IGNORE_BATTERY_OPTIMIZATIONS` | 推荐 | 引导用户允许后台运行/忽略电池优化 |
| `SCHEDULE_EXACT_ALARM` / `USE_EXACT_ALARM` | 推荐 | 提升定时暂停和自动恢复在长时间后台场景下的可靠性 |
| `RECEIVE_BOOT_COMPLETED` | 推荐 | 系统重启或应用更新后自动恢复计时器运行状态 |
| `REQUEST_INSTALL_PACKAGES` | 仅应用内更新需要 | 下载新 APK 后触发系统安装流程 |
| `INTERNET` | 仅更新检查和 DLsite 信息获取需要 | 检查 GitHub Releases、下载更新、获取 DLsite 元数据 |

## 项目结构

```text
lib/
  i18n/                         中/日/英多语言文案
  models/                       MusicTrack、LibraryNode、LibraryEntry、PlaybackMode、作品详情模型
  providers/                    AudioProvider 门面与功能拆分
  screens/                      音频库、播放列表、计时器、设置、视频转音频、DLsite 信息确认

```

## 发行说明 v0.8.8

- **后台播放稳定性**：禁用 ExoPlayer 内置音频焦点管理，防止系统提示音等短暂焦点丢失导致熄屏播放中断；新增 5 秒 grace period 防止曲目切换/缓冲间隙触发前台服务停止；`PlaybackKeepAliveService` 在纯播放时也持有冗余 WakeLock。
- **音量增益扩展**：音量上限从 120% 提升至 200%（约 +6 dB），100%–200% 区间通过 `LoudnessEnhancer` 实现可感知的增益放大。
- **排除状态持久化修复**：排除状态改以 SQLite 为权威来源，重启应用后自动从数据库恢复，不再依赖 SharedPreferences 缓存。
- **刷新性能优化**：`File.existsSync()` 改为后台 isolate 异步批量检查；`_rebuildLibraryIndexes` 引入 O(1) 路径索引，消除 `addOrReplaceTracks` 的 O(n²) 线性扫描；`isLibraryPathExcluded` 改用 `Set.contains` O(1) 查找；批量删除改为单条 `IN` 语句。
- **计时器 UI 优化**：计时结束后（无自动恢复）计时器恢复原样；有自动恢复时胶囊显示自动恢复倒计时；自动恢复播放后计时器重置。
- **元数据备份改进**：单独导入的音频文件详情现在也保存到同目录的 `nameless-audio.json`，与曲库根目录的保存方式一致；同目录多个文件以数组格式共享一个备份文件。
- **通知控制优化**：关闭"播放通知控制"后彻底移除通知（包括无按钮的最小前台通知）；关闭字幕时同步关闭全局字幕显示。
- **文件夹重命名修复**：修复单独导入的文件夹重命名失败（提示"文件夹被占用"）的问题；重命名后封面图片和备份文件正常加载。
- **库加载优化**：启动时减少重复的 `_rebuildLibraryIndexes` 调用；`_loadLibrary` 和 `_loadLibraryEntries` 合并为一次重建。
