# Nameless Audio

Nameless Audio 是面向 Android 的本地音频库、ASMR.ONE 在线播放、多会话播放、字幕、睡眠计时和视频转音频工具。界面、状态管理和数据持久化由 Flutter 承载；实际播放、MediaSession、前台服务、通知、WakeLock、定时器闹钟、文件访问和权限跳转由 Android 原生链路兜底。

当前版本：`0.9.2+902`
最新发布页：[v0.9.2](https://github.com/NameIess-art/nameless-audio/releases/tag/v0.9.2)
许可证：[MIT](https://github.com/NameIess-art/nameless-audio/blob/main/LICENSE)
隐私说明：[PRIVACY.md](https://github.com/NameIess-art/nameless-audio/blob/main/PRIVACY.md)

> 注意：自 `v0.8.0` 起，Android `applicationId` 已从 `com.example.music_player` 更改为 `com.nameless.audio`。旧包名版本不能直接覆盖安装到新版本上，升级前请先备份数据并重新安装。

## 主要功能

### 播放与后台稳定性

- 多会话播放：同时创建多个独立播放会话，每个会话可单独控制播放、暂停、进度、循环模式、音量、左右声道调换和字幕显示。
- 原生播放核心：实际播放集中在 Android `NativePlaybackService`，由 Media3 / ExoPlayer、MediaSession、原生队列、前台服务和 WakeLock 负责后台播放。
- 原生播放队列：非单曲循环模式会把当前文件夹或跨文件夹队列下沉到 ExoPlayer，息屏后仍可继续顺序、随机、上一首和下一首。
- 熄屏稳定性：播放中保持最小前台通知、MediaSession、WakeLock 和 foreground media playback service；通知样式偏好不会停止必要前台服务。
- 音频焦点恢复：原生处理来电、其他媒体抢占、临时焦点丢失和焦点重新获得，`v0.9.0` 修复了部分设备熄屏临时暂停后必须亮屏才恢复的问题。
- 循环模式：支持单曲循环、当前文件夹顺序、当前文件夹随机、跨文件夹顺序、跨文件夹随机。
- 音量增益：每个会话独立音量，支持超过 100% 的增益，Android 端使用 `LoudnessEnhancer` 放大。
- 左右声道调换：适合特定 ASMR 音源，可按会话开启。

### ASMR.ONE 在线播放与下载

- 分类浏览：支持销量、评价、发售、收录、推荐、收藏和历史分类。
- 内容语言：ASMR.ONE 内容可跟随中文、日文、英文语言偏好读取和展示，并在分类、搜索、详情和收藏状态间保持一致。
- 搜索：可搜索作品名、标签、声优、社团和 RJ 号，支持多关键词分隔。
- 分页刷新：分类页支持下拉刷新和上拉加载更多；无更新时保持当前列表，避免刷新时闪烁。
- 登录与收藏：可登录 ASMR.ONE，读取收藏列表，并添加或取消收藏作品。
- 作品详情：展示 RJ 号、社团、标签、声优、发售日、时长、销量、评分、评论数、年龄分级、语言版本和简介。
- 在线添加播放：可读取作品音频树，把单个文件或文件夹节点加入播放列表。
- 字幕匹配：ASMR.ONE 作品会把远端字幕文件匹配到对应音频，支持标题缺少后缀、双扩展名等常见命名方式。
- 下载作品：可选择作品文件树中的文件或文件夹，保留原目录结构下载到本地；音频、字幕、图片、文档等文件节点都会按选择范围处理。
- 下载冲突策略：目标路径存在同名项时可选择跳过或覆盖。
- 下载任务管理：显示准备、下载、完成、失败、已跳过、失败数量、总大小和实时进度；支持取消任务并清除已下载内容。
- 下载元数据备份：下载作品时会生成 `nameless-audio.json`，便于导入本地曲库后恢复作品信息。

### 本地音频库

- 三种导入方式：导入文件夹、导入曲库、导入独立文件。
- Android SAF 优先：优先使用系统文档选择器导入 `content://` 资源并持久化授权，降低对完整文件管理权限的依赖。
- 层级媒体库：根据文件树生成层级节点，适合管理大量子文件夹。
- 刷新监听目录：下拉刷新会重新扫描已导入目录，补齐新增文件并移除磁盘中已删除的条目；未变化的扫描结果会跳过曲库树重建和数据库写入。
- 扫描进度：显示当前文件夹、已发现数量、重复数量和失败数量。
- 搜索与高亮：按名称搜索并显示匹配数量。
- 自然排序与手动排序：文件和文件夹按自然顺序排列，根目录支持拖拽排序。
- 曲库编辑：查看已导入曲库结构，排除或恢复文件夹和单曲，排除状态持久化到 SQLite；从文件管理器删除文件夹后，编辑页会重新同步磁盘快照并清理空壳条目。
- 左滑操作：支持移除、排除和打开详细信息。
- 分类浏览：按全部、标签、声优、社团浏览；分类项可长按复制名称。
- 临时文件导入：单独导入的文件不会绑定到监听目录，直到手动移除。

### 作品信息、DLsite 与封面

- 作品详细信息：编辑文件或文件夹名称、RJ 号、作品标题、社团、声优和标签。
- DLsite 元数据：可按 RJ 号优先获取作品信息；RJ 号缺失时按文件名、文件夹名或作品标题相似匹配，确认后写入。
- DLsite 读取兜底：按所选语言请求 `maniax`、`home`、`girls`、`bl`、`books`、`pro` 等站点的产品 JSON；失败时回退日文、HTML 页面解析和标题搜索，并对限流或临时错误进行重试。
- 元数据备份与恢复：详细信息自动备份到同目录 `nameless-audio.json`，数据库丢失时可恢复。
- 按标题重命名：可用作品标题重命名单个文件或文件夹，并同步曲库、封面、备份和播放会话引用。
- 自动 RJ 预填：导入文件夹时尝试从名称中提取 RJ 号。
- 自动封面发现：优先查找 `cover`、`folder`、`front`、`album`、`artwork`、`poster` 等常见图片。
- 视频帧封面：视频媒体可提取视频帧作为封面候选。
- 文件夹封面选择：在详细信息页左右滑动候选图片，停留后自动应用并全局同步。
- 封面预热：切换会话详情时预热相邻封面，减少闪烁。

### 播放列表、会话详情与字幕

- 会话列表：支持拖拽排序、左滑移除、暂停全部和清空全部。
- 底部播放卡片：可选显示常驻底部卡片，同步封面、进度、字幕和播放状态。
- 会话详情页：以封面作为模糊背景，提供进度条、快进快退、循环模式、音量、计时器、字幕和声道控制。
- 轨道切换：在会话详情页直接切换当前文件夹或分组内的其他音轨，无需新建会话。
- 字幕格式：支持 `.srt`、`.ass`、`.ssa`、`.vtt`、`.lrc`。
- 会话内字幕：会话详情页和播放卡片可显示当前字幕文本。
- 全局悬浮字幕：可跨页面或后台显示字幕，支持按会话开关。
- 字幕悬浮窗样式：支持字体、字号、字体颜色、背景颜色、背景透明度、背景模糊和边框深度，并提供实时预览。
- 字幕位置持久化：全局悬浮字幕位置按会话保存。

### 睡眠计时器

- 两种启动模式：立即开始倒计时，或等待播放开始后触发倒计时。
- 到点暂停：倒计时结束后由 Android 原生链路暂停所有相关会话。
- 自动恢复：可在指定本地时间恢复被计时器暂停的会话。
- 原生闹钟与恢复：使用精确定时能力提升长时间后台、息屏和应用重启后的可靠性。
- 可靠性检查：计时器页检查通知权限、精确定时权限和后台运行条件，并提供跳转修复入口。
- 顶部胶囊状态：播放列表顶部显示计时器倒计时和自动恢复状态。

### 通知、权限与应用设置

- 统一播放通知：支持单会话和多会话通知样式，提供播放、暂停、上一首、下一首、恢复通知和清空通知。
- 富通知开关：关闭的是富播放通知样式；播放中仍保留系统前台通知以保证后台稳定。
- 后台运行引导：可跳转系统电池优化、后台运行、通知、悬浮窗、精确定时和未知来源安装设置。
- 深色模式：支持跟随系统或手动切换。
- 多语言：支持中文、日文和英文界面。
- 多线程播放开关：可控制是否允许多个活跃播放会话同时存在。
- 添加后自动播放：从音频库添加音频时可立即以播放状态创建会话。
- 缓存管理：可清理封面、视频帧、更新包、下载临时文件等缓存，并设置最大缓存上限。
- 应用内更新：从 GitHub Releases 检查、下载并触发系统安装 APK。
- Tab 回顶：点击当前激活的 Tab 图标时，列表自动滚回顶部。

### 视频转音频

- 支持格式：输出 `mp3`、`aac`、`ogg`、`wav`、`flac`。
- 码率设置：MP3 / AAC / OGG 可设置码率；WAV / FLAC 使用格式内置编码参数。
- 转换进度：显示实时进度，支持转换中取消。
- 输出防覆盖：自动避开与现有文件重名的输出路径。

## 导入方式

音频库页右上角 `+` 菜单提供三种导入方式：

| 方式 | 适合场景 | 会导入什么 | 后续刷新行为 |
|---|---|---|---|
| 导入文件夹 | 监听一个固定目录 | 扫描该目录及子目录中的支持媒体文件，并加入监听列表 | 下拉刷新时重新扫描该目录 |
| 导入曲库 | 一个大目录下每个一级子目录是一套内容 | 把下一层子目录分别加入监听；根目录散落媒体也会导入 | 下拉刷新时同步整套曲库结构，并保留排除规则 |
| 导入文件 | 临时加入几个独立媒体文件 | 仅导入手动选中的文件，不绑定监听目录 | 不参与文件夹扫描，直到手动移除 |

## 支持的媒体格式

| 类型 | 格式 |
|---|---|
| 音频 | `flac`、`wav`、`mp3`、`m4a`、`aac`、`ogg`、`opus`、`3gp` |
| 视频，可播放或转音频 | `mp4`、`mkv`、`webm`、`mov`、`m4v`、`avi`、`3gp` |
| 字幕 | `.srt`、`.ass`、`.ssa`、`.vtt`、`.lrc` |

## 权限说明

| 权限 | 是否必须 | 用途 |
|---|---|---|
| `READ_MEDIA_AUDIO` / `READ_EXTERNAL_STORAGE` | 视机型而定 | 兼容直接文件系统扫描与旧式文件选择流程 |
| `MANAGE_EXTERNAL_STORAGE` | 否 | 仅用于需要完整文件系统访问的机型或路径，不是 SAF 导入的前提 |
| `POST_NOTIFICATIONS` | 推荐 | 播放通知、后台控制和状态提示 |
| `SYSTEM_ALERT_WINDOW` | 仅全局字幕需要 | 允许悬浮字幕跨页面或后台显示 |
| `FOREGROUND_SERVICE` / `FOREGROUND_SERVICE_MEDIA_PLAYBACK` | 是 | 后台与息屏播放 |
| `WAKE_LOCK` | 是 | 降低息屏后 CPU 过早休眠导致播放中断的概率 |
| `REQUEST_IGNORE_BATTERY_OPTIMIZATIONS` | 推荐 | 引导用户允许后台运行或忽略电池优化 |
| `SCHEDULE_EXACT_ALARM` / `USE_EXACT_ALARM` | 推荐 | 提升定时暂停和自动恢复可靠性 |
| `RECEIVE_BOOT_COMPLETED` | 推荐 | 系统重启或应用更新后恢复计时器状态 |
| `REQUEST_INSTALL_PACKAGES` | 仅应用内更新需要 | 下载新 APK 后触发系统安装流程 |
| `INTERNET` | 在线功能需要 | ASMR.ONE、DLsite、GitHub Releases 检查更新和下载 |

## 项目结构

```text
lib/
  i18n/                         中文、日文、英文文案
  models/                       MusicTrack、LibraryNode、PlaybackSession、ASMR 和作品信息模型
  providers/                    AudioProvider 门面与功能拆分
  screens/                      ASMR.ONE、本地音频库、播放列表、计时器、设置、视频转音频、DLsite 确认
  services/                     SQLite、Native 桥接、更新、字幕、通知、权限、DLsite、ASMR 下载
  theme/                        主题与深色模式
  widgets/                      通用组件与业务组件
android/app/src/main/kotlin/    原生播放、通知、文件访问、计时器闹钟、保活服务、字幕悬浮窗
third_party/audio_service/      项目内维护的 audio_service fork
test/                           数据库、Provider、通知、计时器、播放桥接、ASMR 下载等测试
```

### Release 构建

Release 构建需要签名密钥。仓库当前保留了本地验证用的回退逻辑：未配置 `key.properties` 时，会使用 debug signing 生成 release 模式 APK；正式发布前建议配置 release keystore。

```bash
flutter build apk --release --split-per-abi
```

该命令会生成三个 APK：

- `app-arm64-v8a-release.apk`
- `app-armeabi-v7a-release.apk`
- `app-x86_64-release.apk`
